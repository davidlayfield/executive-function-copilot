# Task Triage and Prioritization

Convert mess into a list a tired person could act on.

## Bucketing

Sort every item into exactly one of:

- **Delete** — does not matter, will not matter, or already handled.
- **Delegate** — someone else owns this; capture who and what to send them.
- **Defer** — real but not now; capture when it returns and where it lives.
- **Do** — keep on the active list.

## For "Do" items

- **Rewrite as a verb-first next physical action.** "Insurance" → "Email Karen the new policy number."
- **Tag urgent vs important.** Urgent and important first. Important-not-urgent second. Urgent-not-important — defend against these; they are usually other people's emergencies.
- **Project vs task.** A task can be done in one session. A project needs more than one task. If it is a project, define the next action *and* the desired outcome.
- **Time estimate.** Honest: 5m, 15m, 30m, 60m, half-day, day+. People with executive-function challenges chronically underestimate; if in doubt, double it.
- **Dependencies.** Mark anything blocked on a person, decision, or piece of information. Surface these as a "Waiting on" list.

## Output shape

```
DELETE
- ...

DELEGATE → who
- ...

DEFER → when / where
- ...

DO — today
- [must]   <verb-first action> (~Xm)
- [should] <verb-first action> (~Xm)
- [could]  <verb-first action> (~Xm)

DO — this week
- ...

WAITING ON
- <person/decision> for <what>

PROJECTS (need their own plan)
- <name> → outcome: ...
```

## Heuristics

- If something has been on the list more than two weeks untouched, it is either a project, blocked, or not actually wanted. Name which.
- If the list has more than ~7 "Do today" items, it is a wish list. Force a cut.
- If every item is "important," nothing is. Force one must-do.
