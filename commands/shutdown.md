Run an end-of-day shutdown. The goal is permission to stop, not another planning session.

**Process:**
1. Pull what's known — today's plan if one was made, calendar items completed/missed, anything the user just said.
2. If essential context is missing, ask **one** short question: *"Quick — what got done today (a couple of bullets is fine)?"*

**Output shape:**

```
SHUTDOWN — <date>

What got done
- ...
- ...
- ...

What's still open
- <thing>: now lives in <task manager / file / inbox / sticky / your head — name where>

First task for tomorrow
- One specific verb-first action. Pre-staged if possible (file already open, doc created, email drafted).

Permission to stop
- You're done for today. The list is held. You don't have to keep thinking about it.

Reflection (one question, optional)
- Pick one based on the day:
  - "What was harder than expected, and what does that tell you?"
  - "Where did you surprise yourself, even slightly?"
  - "What did you avoid, and is it actually still important tomorrow?"
```

**Rules:**
- Every "still open" item must have a *location*. Open loops in your head are not held. Move them to a file, a task manager, or paper.
- Do not produce tomorrow's full plan here. One first task is enough; full planning is for /daily-plan.
- End with the permission line. Do not append more tasks.
- No moralizing about what didn't get done. The day is the day.

If a `personal-operating-manual.md` exists and the day produced a durable insight (e.g., "morning meetings kill my deep-work day"), propose adding 1–2 lines to it. Show the diff. Wait for approval.
