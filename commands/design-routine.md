---
description: Design a routine that survives bad days — minimum / normal / stretch versions, plus a restart rule.
argument-hint: <routine name or what you want to build>
---

Apply the `routine-and-habit-design` skill.

**Process:**
1. Take the routine name from the argument. If unclear, ask **one** question: *"What's the routine for, and when do you want it to happen?"*
2. Read the `personal-operating-manual.md` if available — honor stated energy patterns, derailers, and reminder preferences.
3. Build the routine for the **bad day**, not the ideal day.

**Output shape:**

```
ROUTINE: <name>

Trigger
- When it starts. Anchor to an existing behavior if possible. ("After I pour coffee.")

Minimum viable (the "I feel awful" version)
- 1–3 minutes. Counts as success on bad days.

Normal
- The realistic version. Honest about time.

Stretch
- High-energy version. Optional. Never the standard.

Environment setup
- What gets staged in advance, what gets moved out of the way.

Reminder strategy
- One channel only. Where it lives. (Phone, paper, calendar, partner — not all of them.)

Restart rule
- After a missed day: run minimum.
- After a missed week: redesign — do not push.

Tracking
- One low-friction signal. Not a spreadsheet.

First run
- When are you running this for the first time? What's pre-staged?
```

**Rules:**
- The minimum version must feel insultingly easy. That is its job.
- No 12-step morning routines. No new apps unless the user asks.
- One routine at a time. If the user asks for three, build one and queue the others.
- Re-design every 4–6 weeks; surface that in a closing line.

If the routine is durable, propose adding it to the `personal-operating-manual.md` under "Routines I run." Wait for approval.
